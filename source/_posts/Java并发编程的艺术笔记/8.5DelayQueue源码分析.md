# 8.5 DelayQueue 源码分析

## 1. 数据结构

```java
public class DelayQueue<E extends Delayed> extends AbstractQueue<E>
    implements BlockingQueue<E> {

    private final PriorityQueue<E> q = new PriorityQueue<E>();
  
    private final transient ReentrantLock lock = new ReentrantLock();
  	// 出队线程阻塞在这里。
    private final Condition available = lock.newCondition();
  
    private Thread leader = null;

    /**
     * Condition signalled when a newer element becomes available
     * at the head of the queue or a new thread may need to
     * become leader.
     */

    public DelayQueue() {}
}

public interface Delayed extends Comparable<Delayed> {

    // 返回还要延迟多久，
  	// 小于等于 0 时，说明时间到了。
    long getDelay(TimeUnit unit);
}
```

1. 使用 `PriorityQueue` + `ReentrantLock` 实现。
2. 内部元素必须实现 `Delayed` 接口，等待队列的元素是按照节点的  `compareTo()` 从小到大出队的。
3. 使用 `leader` 每次头结点变化时，唤醒一个元素，使其有超时等待，其他线程无限等待。

## 2. 入队 offer(e)

```java
// 无界，所以 put(e) 也是直接调用 offer(e)。
public boolean offer(E e) {
    final ReentrantLock lock = this.lock;
  	// 1. 加锁。
    lock.lock();
    try {
      	// 2. 入优先队列。
        q.offer(e);
      	// 3. 如果这次入队的元素最小，有了赋予一次 leader 的机会，唤醒等待队列上的一个线程。
        if (q.peek() == e) {
            leader = null;
            available.signal();
        }
        return true;
    } finally {
      	// 4. 解锁。
        lock.unlock();
    }
}
```

## 3. 阻塞出队 take()

```java
public E take() throws InterruptedException {
  	// 1. 加锁。
    final ReentrantLock lock = this.lock;
    lock.lockInterruptibly();
    try {
        for (;;) {
            E first = q.peek();
          	// 2. 如果队列为空，阻塞。
            if (first == null)
                available.await();
            else {
                long delay = first.getDelay(NANOSECONDS);
              	// 3. 如果首节点过期，返回。
                if (delay <= 0)
                    return q.poll();
                first = null; // don't retain ref while waiting
                // 4.1 如果 leader 不为 null，说明其他线程抢到了这次赋予 leader 的机会。
              	//		例如本线程是等待队列里被唤醒的，但是赋值 leader 前，
              	//		有了一个新的 take() 抢先完成了赋值 leader 的机会。
              	if (leader != null)
                    available.await();
                else {
                  	// 4.2 得到了赋值 leader 的机会，不忙等、而是等待 delay 时间。
                    Thread thisThread = Thread.currentThread();
                    leader = thisThread;
                    try {
                        available.awaitNanos(delay);
                    } finally {
                      	// 5. 清空 leader。
                        if (leader == thisThread)
                            leader = null;
                    }
                }
            }
        }
    } finally {
      	// 6. 如果本线程取出一个元素后，队列里还有元素，
      	//			则又有赋予 leader 的权限了。唤醒一个线程。
        if (leader == null && q.peek() != null)
            available.signal();
      	// 7. 释放锁。
        lock.unlock();
    }
}
```

- 在以下两个情况，会有一次赋值 `leader` 的机会，唤醒一个等待队列上的出队线程。
  - 在入队操作后，如果本次入队的是最小元素。
  - 出队操作后，如果队列还有元素。
- 争夺到 `leader` 机会的出队线程，进行有超时的等待。
- 队列没有元素时，所有出队线程都是无限等待。

## 4. 非阻塞出队 poll()

```java
public E poll() {
    final ReentrantLock lock = this.lock;
  	// 1. 加锁。
    lock.lock();
    try {
        E first = q.peek();
      	// 2.1 如果队列为空，返回 null。
        if (first == null || first.getDelay(NANOSECONDS) > 0)
            return null;
      	// 2.2 否则，出队首元素。
        else
            return q.poll();
    } finally {
      	// 3. 解锁。
        lock.unlock();
    }
}
```
