1. 介绍

基于数组的最小堆

锁

  • 使用 ReentrantLock lock 非公平锁，在所有出队、入队操作前后都要使用锁。
  • 使用 Condition notEmpty 作为 take() 操作时队列为空的等待队列。
    • 在入队操作后，唤醒 notEmpty 上的一个线程。
    • 因为队列无界，不需要 notFull，总是能填进去，除非到达最大值 MAX_ARRAY_SIZE。
  • 扩容的如下操作，避免扩容时，不能出队。
    1. 先释放锁，
    2. 使用 allocationSpinLock 自旋锁创建新的数组。
    3. 再获取锁，拷贝数据。
